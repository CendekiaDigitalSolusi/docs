# Discussion Settings

**Discussion Settings** manages comments on blog articles. This setting becomes **default for new articles**.

Pre-built articles can have their own different settings.

## How to Access

1. Entered into [Admin Dashboard](../../dashboard-admin.md)
2. Open the **Admin menu → Settings → Blog → Discussion Settings**

## Available Settings

### 1. Allow People to Post Comments on New Articles

If **active**: New articles will **accept comments** from visitors
- A comment column/form will be available below the article
- Visitors can leave feedback/responses

If **off**: New articles **do not accept comments** by default
- The comments column will be turned off
- Visitors cannot post comments

:::info
Changes here only apply to **new articles**. Old articles still follow their old settings. You can change per article if necessary.
:::

### 2. Require Users To Log In Before Posting Comments

If **active**: Only logged in users can post comments
- Effective for reducing spam from bots/anonymous
- More controlled in terms of moderation

If **off**: Anyone can post comments (if there is no other verification)
- More open, but prone to spam

:::warning
For public school websites, it is best to activate this setting to reduce spam and inappropriate comments.
:::

### 3. Close Old Article Comments Automatically

If **on**: Comments will be automatically closed for articles that have reached a certain age limit.

**Related settings:**
- **Close comments on articles older than (X) days**
- Fill in the number of days

**Example:**
- If filled in`14`, articles created more than 14 days ago are not accepting new comments
- Old comments are still visible, but new comments cannot be added

**Suggested value:**
- `7–14`a day for fast-changing news/announcements
- `30–90`day for evergreen/perennial articles (such as guides, FAQs)
- `0`or inactive if you want all articles to still receive comments

:::warning
This feature is great for reducing spam on old articles. But make sure it doesn't interfere with discussions that are still relevant and need feedback.
:::

## Tips

- **Comment moderation** - Enable moderation so that comments do not appear immediately without review
- **Balance between open & safe** - Allow comments but with login verification for security
- **Set time limits wisely** - Don't close comments too quickly on important articles
- **Regular comment reviews** - Delete spam and inappropriate comments to maintain discussion quality
